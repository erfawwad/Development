How JavaScript Runs Your Code

When you run JavaScript, the engine builds an environment called the Execution Context. Explore the two phases of execution, the memory heap, and the Call Stack below.

Core Concepts

The building blocks of the JavaScript runtime.

✦ Global Context

GEC

The default environment created when your script starts. There is only one Global Execution Context.

✴ Function Context

FEC

Created every time a function is invoked. Each call gets its own brand-new context with private memory.

⚙ Two Phases

Create & Execute

First, memory is allocated (Hoisting). Then, code runs line-by-line assigning values and executing logic.

☰ Call Stack

LIFO Principle

Last In, First Out. Functions are pushed when called and popped when they return.

Creation vs. Execution

Understanding the difference between the two phases.

Feature Phase 1: Memory Creation Phase 2: Code Execution
Variables Allocated memory. var set to undefined. Assigned actual values (e.g., 5, "hello").
Functions Entire function code is stored in memory. Function logic runs if invoked.
Stack Activity Context is Pushed (prepared). Logic is processed within the context.
Analogy "Planning / Setup" "Doing / Action"

Summary

The JavaScript engine manages complexity through Execution Contexts. By separating Memory Creation from Execution and using a Call Stack, it ensures orderly, single-threaded processing of your code.
