HTML5 Mastery Guide

The fundamental building blocks of the web. A comprehensive roadmap from basic syntax to semantic SEO and accessibility.

1. Boilerplate & Meta Tags

Every HTML document starts with this essential structure to define character encoding and responsiveness.

<!DOCTYPE html> <!-- Tells browser we are using HTML5 -->
<html lang="en">
<head>
    <meta charset="UTF-8"> <!-- Supports almost all characters -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>

2. Semantic Elements (SEO & Accessibility)

Stop using <div> for everything. Semantic tags tell the browser and screen readers what the content is.

<header>

Introductory content or navigation links.

<nav>

Navigation blocks (menus, sidebars).

<main>

The primary content of the document.

<section>

Thematic grouping of content, usually with a heading.

<article>

Self-contained content (blog post, news story).

<aside>

Content indirectly related to the main content (sidebar).

<footer>

Contact info, copyrights, or sitemaps.

<figure>

Self-contained content, like photos with <figcaption>.

3. Text Formatting & Hierarchy

Headings

Use <h1> only once per page. Use <h2> through <h6> for sub-levels.

Block vs. Inline

  • Block: p, div, h1, ul, li (Takes full width)
  • Inline: span, a, strong, em, img (Takes only needed width)

Lists

Unordered (ul), Ordered (ol), and Description (dl).

4. Links, Images & Media

Hyperlinks

<a href="url" target="_blank" rel="noopener">Link</a>

Use rel="noopener" for security with target="_blank".

Images (Responsive)

<img src="path" alt="Descriptive text" loading="lazy">

alt is mandatory for accessibility. loading="lazy" improves performance.

5. Form Elements & Attributes

Input Types

text password emailId number checkbox radio date file range color

Validation Attributes

  • required - Field must be filled
  • minlength / maxlength - Text length
  • min / max - Numeric range
  • pattern - RegEx validation
  • placeholder - Hint text

Best Practice: Labels

Always wrap or link labels using the for and id attributes to increase click area and help screen readers.

6. Accessibility (ARIA)

When semantic HTML isn't enough, use ARIA (Accessible Rich Internet Applications) attributes.

role="..." - Defines what an element is (e.g., role="button").
aria-label="..." - Provides a string for screen readers if no text is visible.
aria-hidden="true" - Hides decorative elements from screen readers.
aria-expanded="..." - Indicates if a menu or accordion is open.
